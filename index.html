About the Center

Built for Industry.
Backed by Research.

The PTC is BITS Pilani WILP Division's dedicated penetration testing, cybersecurity and system validation facility for automotive, IoT and healthcare applications. Established under a formal MOU with Bosch Global Software Technologies, the center bridges the gap between academic knowledge and real-world security testing — operating on live industrial platforms, not simulations.

With 23 lab modules actively commissioned and staffed by 5 student interns and 2 research assistants, the center delivers certified, production-grade security validation for the automotive and connected systems industries.

All projects address live vulnerability categories in automotive or connected systems, published or presented at international IEEE-affiliated venues.

01
Knowledge Distillation for Lightweight ECU Fingerprinting in CAN Bus Security
Automotive Security AI / ML Embedded Systems XGBoost TFLite Scikit-learn
98%
Accuracy
Modern vehicles contain dozens of ECUs communicating over the CAN bus. This project developed an AI-based fingerprinting system that monitors unique voltage signatures of each ECU — enabling real-time detection of hardware spoofing attacks. Knowledge distillation was applied to compress the model into a lightweight TFLite neural network suitable for resource-constrained microcontrollers, achieving 97% accuracy in the distilled version.
Dataset54 ECUs across 10 vehicles (public dataset)
MethodXGBoost One-vs-Rest + SMOTE + Knowledge Distillation
PublishedIEEE Xplore — ICNGN 2025, Singapore
AuthorsSutirth Rath & Ramya K
02
Explainable ML + LLM Framework for Anomaly Detection in AWS CloudTrail Logs
Cloud Security LLM / NLP Threat Detection AWS XGBoost Streamlit
F1
0.60 Score
This project combines supervised ML for anomaly detection with a large language model to generate plain-English explanations for flagged events — bridging automated detection and human analyst judgment. The integrated pipeline significantly reduces analyst workload and is delivered through an interactive Streamlit dashboard. XGBoost outperformed rule-based baselines especially for rare, abnormal events.
ArchitectureXGBoost classifier + LLM explanation pipeline
InterfaceStreamlit dashboard with human-readable threat summaries
PublishedIEEE Xplore (ICNGN 2025) + Poster at SPACE 2025, IIT Guwahati
AuthorsSurya Karthikeya Varma I & Ramya K
03
IoT OTA Firmware Security — DNS Spoofing & Protocol Downgrade Detection
IoT Security Network Security OTA Updates Python Scapy VirtualBox
0
False Positives
Wi-Fi connected IoT devices are vulnerable during over-the-air firmware updates. This work developed a lightweight packet-level detector that identifies TTL/IP anomalies, duplicate DNS responses, and TLS→HTTP fallback in real time — with zero false positives and minimal computational overhead, making it suitable for IoT gateways. Existing cryptographic protections are too heavy for resource-constrained devices.
TestbedControlled VirtualBox environment with live OTA simulation
AttacksDNS spoofing, TLS→HTTP protocol downgrade, normal OTA
PublishedPoster — SPACE 2025, IIT Guwahati
AuthorsGautham Nair & Ramya K
04
Dual-Lock Secure Bootloader — Cryptographic Verification & UART Deinitialization
Embedded Security Cryptography Secure Boot C/C++ ARM MCU SHA-256 RSA
60s
Debug Window
This proof-of-concept introduces a "dual-lock" bootloader design — on cryptographic failure (SHA-256 + RSA verification), the system performs immediate hardware-level UART deinitialization to prevent brute-force or replay attacks. On successful verification, it opens a precisely timed 60-second controlled debug shell before sealing the system again, giving authorized developers a narrow, safe diagnostic window.
SecuritySHA-256 hash + RSA signature verification before execution
Failure ModeHardware-level UART deinitialization — prevents replay/brute-force
PublishedPoster — SPACE 2025, IIT Guwahati
AuthorsAdvait Deshmukh & Ramya K
05
Cyber-Resilient AI Thermal Management for EV Lithium-Ion Batteries — Adversarial Attack Analysis
EV Security Deep RL Hardware-in-the-Loop TensorFlow PyTorch HIL Edge OpenAI Gym
26°C
vs 29.9°C PID
A Predictive Deep Reinforcement Learning agent on a Hardware-in-the-Loop edge controller uses GPS route data to anticipate upcoming thermal stress and proactively pre-cool the battery before high-load events. The system was analyzed for resilience against adversarial attacks — a critical concern given AI-controlled safety systems in EVs are high-value targets. The DRL agent maintained batteries within the 26.0°C safety limit while PID allowed a 29.9°C overshoot.
ControllerPredictive DRL agent vs PID and MPC baselines
PlatformHardware-in-the-Loop (HIL) edge controller with GPS integration
AdversarialResilience tested against PGD-based adversarial attack scenarios
ResultDRL outperforms MPC; MPC already outperforms PID baseline

Industry Collaboration · Bosch Cyber Security Team
aRNA-Based Adversarial Attack on UWB Jammer Localization Model
A joint academic-industrial research effort between the PTC and Bosch's active cyber security team. The work evaluates a PGD-based aRNA adversarial attack against a domain-adapted ConvNeXt-1D CIR localization model used in Ultra-Wideband jammer detection — a critical system for automotive security.
